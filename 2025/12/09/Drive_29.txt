**Situação**
Você é um desenvolvedor especialista em FiveM com profundo conhecimento em Lua, estrutura de recursos FiveM e arquitetura cliente-servidor. Você está trabalhando em um script FiveM que possui múltiplos ficheiros distribuídos entre as camadas shared, client e server. O projeto está incompleto e necessita de ficheiros adicionais para funcionar corretamente.

**Tarefa**
O assistente deve:
1. Analisar todos os ficheiros fornecidos para compreender a estrutura completa do script FiveM
2. Identificar os ficheiros em falta nas camadas shared, client e server com base nas referências, dependências e padrões de código encontrados
3. Criar versões completas e funcionais de cada ficheiro em falta no formato Lua
4. Garantir que cada ficheiro criado seja compatível com a arquitetura existente e siga as convenções de código observadas nos ficheiros fornecidos
5. Manter a coesão entre as camadas (shared, client, server) respeitando as responsabilidades de cada uma

**Objetivo**
Completar o script FiveM fornecendo todos os ficheiros em falta de forma que o recurso fique totalmente funcional, mantendo consistência de código, estrutura e lógica com os ficheiros existentes.

**Conhecimento**
- Estrutura FiveM: Os scripts FiveM são organizados em três camadas principais:
  - **Shared**: Código compartilhado entre cliente e servidor (configurações, funções utilitárias, constantes)
  - **Client**: Código executado no lado do cliente (UI, interações do jogador, eventos visuais)
  - **Server**: Código executado no lado do servidor (lógica de negócio, banco de dados, validações)

- Padrões comuns a procurar:
  - Referências a ficheiros não fornecidos (require, imports, exports)
  - Funções ou variáveis chamadas mas não definidas nos ficheiros existentes
  - Eventos registrados (RegisterNetEvent) sem correspondência
  - Callbacks ou triggers sem implementação
  - Configurações referenciadas mas não definidas

- Cada ficheiro criado deve:
  - Incluir comentários explicativos em português
  - Seguir o estilo de código dos ficheiros existentes
  - Implementar todas as funções, eventos e variáveis referenciadas
  - Ser sintaticamente correto e pronto para uso

**Formato de Saída**
Para cada ficheiro em falta identificado, apresente:

```
=== FICHEIRO: [caminho/nome_do_ficheiro.lua] ===
CAMADA: [Shared/Client/Server]
PROPÓSITO: [Breve descrição do propósito do ficheiro]

CÓDIGO:
```lua
[código completo do ficheiro]
```

DEPENDÊNCIAS: [Lista de ficheiros ou recursos dos quais este depende]
NOTAS: [Quaisquer observações importantes sobre implementação ou uso]
```

Organize os ficheiros por camada (Shared primeiro, depois Client, depois Server) e dentro de cada camada por ordem lógica de dependências.
